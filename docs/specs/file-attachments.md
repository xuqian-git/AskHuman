# 需求：AskHuman 提问附带文件（`-f` 文件附件）

> 状态：已实现
> 关联计划：`docs/plans/file-attachments.md`

## 1. 背景

`AskHuman` 既是 CLI 又能弹出 GUI 窗口收集人类回应。当前已支持的「图片」是 **人 → AI** 方向：用户在弹窗里粘贴/拖拽/选择图片，作为回答的一部分回传给 AI（落盘后在 stdout 的 `[图片]` 区块给出路径）。

本需求新增的是 **AI → 人** 方向的能力：AI 在发起提问时，通过命令行附带一个或多个本地文件，弹窗在提问内容下方展示这些文件附件，方便人类查看后再作答。

## 2. 目标

AI 调用示例：

```bash
AskHuman "请看看这个文档有没有问题？" -f ~/Documents/some_file.md -f ./diagram.png
```

弹窗在提问正文下方显示文件附件列表，支持：

- **单击**：选中（高亮）该附件
- **双击 / 回车**：用系统默认程序打开该文件
- **空格**：预览（macOS QuickLook）

## 3. 已确认决策

| 编号 | 决策项 | 结论 |
|---|---|---|
| D1 | 方向与输出契约 | 文件是「提问展示附件」（AI→人），仅在弹窗显示；**不进入 stdout 结果区块**（不新增 `[文件]` 区块），不改变现有 stdout 契约 |
| D2 | 参数形式 | 新增 `-f` / `--file <path>`，可多次出现 |
| D3 | 路径解析 | 支持「绝对路径 / 相对当前工作目录 / `~` 家目录展开」，最终解析为绝对路径 |
| D4 | 文件不存在 | 解析时若某文件不存在 → stderr 报错并以退出码 `1` 终止（与图片落盘失败一致） |
| D5 | 打开文件实现 | 自定义 Rust 命令 + 各平台原生打开方式（macOS `open` / Windows `start` / Linux `xdg-open`），**不引入新插件** |
| D6 | 交互 | 单击=选中高亮（单选，同一时刻仅一个选中）；双击 或 回车=打开；空格=预览 |
| D7 | 空格预览 | macOS 用 `qlmanage -p` 触发系统 QuickLook 预览；Windows / Linux 无统一 QuickLook → 空格回退为「打开」 |
| D8 | 展示位置 | 提问正文（Markdown/纯文本）下方、预定义选项上方，作为独立「附件」区 |
| D9 | 展示样式（非图片） | 每个附件一行：通用「按扩展名图标」+ 文件名 + 文件大小；完整绝对路径作为 tooltip（hover 提示） |
| D10 | 展示样式（图片） | 图片类文件显示**缩略图**，与现有「回答区图片缩略图」一致的方式（base64 data URL + CSS 缩放显示） |
| D11 | 图片判定 | 按扩展名判定图片类：`png / jpg / jpeg / gif / webp / bmp / svg` |
| D12 | 缩略图加载 | 新增 Rust 命令读取本地图片 → 返回 base64 data URL 给前端；后端不做缩放，前端用 CSS 缩放 |
| D13 | Telegram 发送 | Telegram 渠道启用时（GUI+Telegram 抢答、或仅 Telegram 的 headless 路径），在发完提问消息后，把每个 `-f` 文件发到 Telegram |
| D14 | Telegram 发送方式 | 图片类用 `sendPhoto`（可内联预览），其它文件用 `sendDocument`；均带文件名 |
| D15 | Telegram 容错 | 单个文件上传失败 → stderr 警告并跳过（不影响整次提问与其它文件），**同时向 Telegram 发送一条失败提示消息，包含该文件的本地路径**（尽管远端无法访问） |
| D16 | 文档同步 | 同步更新 `--help` 文案、设置页「参考提示词」(`prompts.rs`)、`README` |

## 4. 约束与既有规则（不可破坏）

- **stdout 洁净**：stdout 仍只输出现有结果区块（`[选择的选项]` / `[用户输入]` / `[图片]` / `[状态]`）。文件附件**不**改变 stdout，下游解析契约不变。
- **不引入新 Tauri 插件**：打开/预览/读取缩略图均用自定义 Rust 命令实现，避免新增 capability 配置与依赖体积。
- **现有功能契约不变**：多 channel「抢答」、回答区图片上传、配置文件、`--settings`/`--help`/`--version`、退出码语义（0/1/3）均保持。
- **release 构建模式**：生产构建须启用 `tauri/custom-protocol`（见仓库已修复项），本需求不回退该修复。

## 5. 验收标准

1. `AskHuman "..." -f <存在的文件>` 弹窗在提问下方显示该附件；非图片显示图标+文件名+大小，图片显示缩略图。
2. 多个 `-f` 同时生效，按给出顺序展示。
3. 单击选中高亮；双击或选中后回车，用系统默认程序打开文件；空格在 macOS 触发 QuickLook 预览、在其它平台回退为打开。
4. `-f` 指向不存在的文件 → stderr 报错、退出码 `1`，不弹窗。
5. 文件附件不出现在 stdout 任何区块；回答（选项/文字/图片）行为与现状一致。
6. 配置了 Telegram 时：提问消息后，图片以 `sendPhoto`、其它以 `sendDocument` 发到 Telegram，带文件名；某文件上传失败时，stderr 有警告且 Telegram 收到一条含该文件路径的失败提示，其余文件与提问流程不受影响。
7. `--help`、参考提示词、README 均反映 `-f` 用法。

### 2026-06-03 实现期迭代调整

- **空格预览改为原生 QLPreviewPanel**（替代 D7 的 `qlmanage -p`）：经 objc2 定义 `NSResponder` 子类并插入弹窗响应链，实现 `QLPreviewPanelController`/`DataSource`/`Delegate`，方向键在面板内逐个切换附件，弹窗侧据 `preview-index` 同步高亮；面板关闭经 `preview-closed` 复位。Windows/Linux 仍回退为「打开」。
- **取消快捷键调整**：移除 `Esc` 关闭弹窗（`Esc` 仅用于关预览），改为 `⌘W` 取消、`⌘↵` 发送，并在按钮上标注。
- **`-f` 附件胶囊新增 macOS 原生能力**：
  - **拖出**到其它应用（引入 `tauri-plugin-drag`，拖到 Dock/应用图标用其打开）；拖拽预览用 `NSWorkspace` 取的真实文件图标（光栅化到 64×64）。拖回窗口内的 `-f` 附件按路径匹配忽略，不计入回复。
  - **右键菜单**（原生 `NSMenu`，Finder 风格）：打开 / 打开方式▸（应用列表+图标、其他…）/ 快速查看 / 在访达中显示 / 拷贝文件 / 拷贝路径。
- **新增「回复文件」方向（人→AI）**：弹窗支持拖入非图片文件，作为回复文件附件（输入框下方胶囊、可移除），提交后在 stdout 新增 `[文件]` 区块输出其绝对路径（不复制）。注意这与 D1 不冲突：D1 约束的是 `-f` 提问附件（AI→人）不进 stdout；`[文件]` 属于回复方向，与 `[图片]` 同类。
- **来源名定制**：新增环境变量 `ASKHUMAN_ENV_SOURCE_NAME`，定制弹窗标题与 Telegram 消息头「Question from {名称}」，缺省回退「the Loop」。
- 因引入 `tauri-plugin-drag`，相应放宽「不引入新 Tauri 插件」约束（仅限该拖出能力）；其余打开/预览/读取缩略图仍为自定义命令。

### 2026-07（Quick Look 与作答并行）

- Quick Look 打开后，Popup 内点击、选择 Message、编辑答案、操作编辑器按钮和切题都不主动关闭面板；附件选中高亮继续保留。
- 用户回到 Popup 后，面板稍后手动关闭不得把焦点从 textarea / 当前控件抢回附件；若用户始终在 Quick Look 内操作并关闭，仍可恢复附件焦点。
- 焦点不在 textarea / input 时，选中附件后空格在“打开 / 关闭预览”间切换；输入框内空格始终输入字符。Esc 和面板关闭按钮沿用原生手动关闭。
- 提交、取消或 Popup 销毁时主动关闭预览；仅切题不关闭。
