# 需求：多问题弹窗交互重构（去高亮跳动 + 双模式 + ⌘↵ 修 bug）

> 状态：已确认（待按计划实现）
> 关联计划：`docs/plans/multi-question-interaction-redesign.md`
> 关联既有：`docs/specs/multi-question-vertical.md`（纵向列表基础形态，本需求**取代其 §3.3 高亮/§3.2 当前题语义/§3.4 ⌘↵ 语义**，其余沿用）

## 1. 背景与问题

纵向多问题弹窗（`verticalMode` = 实验开关开 且 n>1）当前用单一「当前题」指针 `current` 驱动
高亮、⌘1–9 目标、⌘ 角标、语音目标。三个问题：

1. **视觉乱 + hover 跳动**：`current` 被 **滚动比例线 / hover 进卡片 / 键盘导航** 三源争夺；`.q-card.active`
   的常驻高亮（`::before` 覆盖整卡且溢出边缘的 accent 底色块 + `::after` 左侧 3px accent 竖条）随 `current`
   在各题间 opacity 淡入淡出——鼠标一划过卡片就整块高亮来回搬家，观感杂乱、"界面一直在跳动"。
2. **蓝条重**：左侧 3px、accent 55% 的常驻竖条偏重。
3. **⌘↵ 跳回已答题**：`⌘↵` = `current+1` 纯顺序前进；先答后面的题、再回头答前面的，按 `⌘↵` 会落到
   **已经答过**的题上。

## 2. 目标（本次只改这几件事）

- **删掉会跳动的常驻高亮**（大色块 + 蓝条），焦点本身**无常驻视觉**。
- **两种操作模式并存、无需"进入模式"**：鼠标点选 / 键盘操作，弹窗一弹出即可用。
- **⌘↵ 永不落到已答题**，按需求推进到"下一个该做的事"。
- **底部提交按钮状态**在"没看完 / 看完仍有未答 / 全部已答"三阶段清晰变化。

**不改**：单问题(n=1) 与旧版顺序模式；提交协议 / 后端聚合；按题作答数组；图片归属 / 语音 / 折叠输入框；
"读完全部题才出现提交"的门槛与底部四钮（取消/上一个/下一个/提交）结构。仅前端 `src/views/PopupView.vue`
（+ 可能一条新 i18n 文案），不引入新依赖。

## 3. 已确认决策

### 3.1 焦点模型（T1）

| 编号 | 决策项 | 结论 |
|---|---|---|
| F1 | 焦点来源 | 保持多源：**hover / 滚动 / 点选项或输入框 / 键盘导航** 均可设焦点（沿用现有机制）。跳动根因是常驻高亮而非多源本身，删高亮即可 |
| F2 | 焦点重视觉 | **删除** `.q-card.active` 的 `::before` 底色块与 `::after` 蓝条（跳动根因）。删后焦点仅剩下方 F3 的轻量指示 |
| F3 | 焦点显露 | **不新增任何视觉**，沿用现状两处：①焦点题上**一直淡显**的 `⌘1–9` 选项角标（随焦点在题间挪、按住 ⌘/Ctrl 变亮，来自 `cardOptionHotkey` + `.cmd-held`，**本次不改**）；②**纯键盘跳转**时整题闪一下（见 3.2）|

> 说明：删大高亮后 hover 改焦点不再产生"重"的视觉搬家（仅剩淡角标随焦点移动，用户已接受），也顺带解决
> "滑到第 3 题、⌘数字却命中第 1 题"的错位——焦点跟随你正在看/操作的那道题。

### 3.2 键盘焦点"闪一下"（T2）

| 编号 | 决策项 | 结论 |
|---|---|---|
| B1 | 效果 | 跳转到某题时，**整题**闪一下 accent 高亮再渐隐（约 0.5s），明确告诉用户键盘焦点落在这题 |
| B2 | 触发范围 | **只有纯键盘跳转**（`⌘↵` / `⌘[` / `⌘]`）才闪。**点『上一个/下一个』按钮、hover、点选项都不闪**（闪只为提示"键盘"焦点） |
| B3 | 可重复 | 连续跳到同一题也应能再次闪（用元素级动画重放，不靠 class 是否变化） |
| B4 | 降级 | `prefers-reduced-motion` 下不闪（或瞬时），跳转仍照常滚动/置焦点 |

### 3.3 ⌘↵ 语义（T4，修 bug）

`⌘↵` 从**当前焦点**出发：

1. 向后（index > 焦点）找**第一个未答**的题 → 跳过去（闪 + 滚入视口 + 置焦点）。
2. 后面没有未答的题了：
   - 若**已看完全部**（门槛满足）→ **提交**；
   - 否则 → 跳到**下一个没看过**的题继续读（推进门槛）。
3. **永不落到已答题**（修 bug）。

- "已答"判定 = `isAnswered(i)`（有勾选 / 有输入 / 有图片 / 有回复文件；严格选择模式即"有勾选"）。
- 自由模式想**留空提交**：阶段②/③ 直接点『提交』按钮（不必逐题填）。

### 3.4 上一个 / 下一个（T4=乙）

| 编号 | 决策项 | 结论 |
|---|---|---|
| N1 | 语义 | 顺序移动（`goRel(∓1)`），用于**按序读题**，可落到已答题 |
| N2 | 快捷键 | `⌘[` = 上一个、`⌘]` = 下一个；键盘触发时**闪**；点同名按钮**不闪** |
| N3 | 边界 | 首题禁用『上一个』、末题禁用『下一个』 |

### 3.5 底部『提交』按钮状态机（T3，核心）

底部四钮结构不变（取消 / 上一个 / 下一个 / 提交）。**挂着 `⌘↵` 的按钮 = `⌘↵` 实际会执行的动作**（沿用现设计哲学，杜绝歧义）。

| 阶段 | 上一个 `⌘[` | 下一个 `⌘]` | 提交 |
|---|---|---|---|
| ① 还有没看过的题（门槛未满） | 首题禁用 | **主按钮**·顺序 +1·末题禁用 | **隐藏** |
| ② 全部看过·仍有未答 | 可用 | 主按钮·顺序 +1·末题禁用 | **出现但次要**（非主按钮、无 `⌘↵`）；旁显『还剩 N 题未答』；`⌘↵` 此时=跳下一未答 |
| ③ 全部已答（严格模式=全选） | 可用 | 末题禁用 | **主按钮高亮 + `⌘↵`**；`⌘↵`=提交 |

- 主按钮规则：阶段③ 主按钮 = 提交；否则（①②）主按钮 = 下一个（其可用时）。
- "全部看过"沿用现门槛（末题进视口 / 一屏全可见即满足）；窗口很高、一弹出就露出全部题时门槛立即满足，可直接提交（阶段②/③）。

## 4. 约束与既有规则（不可破坏）

- 仅前端 `src/views/PopupView.vue`（+ 可能一条 `popup.unansweredRemaining` i18n）。
- 单问题(n=1) / 旧版顺序模式行为逐项保持现状。
- 不动提交协议 / 后端聚合 / 按题作答数组 / 图片归属 / 语音 / 折叠输入框 / 读完门槛。
- 不引入新依赖。

## 5. 验收标准

1. 多问题弹窗**无常驻高亮、无蓝条**；鼠标划过/滚动**不再有高亮搬家或跳动**。
2. 弹窗一弹出即可操作：鼠标可直接点选项/点输入；`⌘1–9`、`⌘↵`、`⌘[`/`⌘]` **无需先按任何键**即刻可用。
3. 焦点题的 `⌘1–9` 角标沿用现状（平时淡显、按住 ⌘ 变亮）；本次不新增 ⌘-held 视觉。
4. `⌘↵` / `⌘[` / `⌘]` 键盘跳转时目标题**闪一下渐隐**；点『上一个/下一个』按钮、hover、点选项**不闪**。
5. `⌘↵` 从焦点向后找下一个未答题；后面无未答→已看完则提交、否则去下一个没看过的题；**绝不落到已答题**（先答后面再答前面的场景不再回跳）。
6. 底部提交按钮按 §3.5 三阶段变化；挂 `⌘↵` 的按钮恒等于 `⌘↵` 的实际动作。
7. 单问题(n=1) 与旧版顺序模式行为与现状完全一致。

## 6. 反馈意见

### 2026-07（首次实现后验收反馈，已修）

用户实测首版后指出以下问题，均已修正：

1. **闪光区域用错**（此前反复调过的高亮区域别再改）：闪光**复用旧版当前题高亮那块区域**
   `.q-card::before`（`inset: 8px -12px -6px -12px` + 圆角 + accent 淡底），仅由「常驻」改为一次性
   `opacity 1→0` 渐隐（`.flash` 类 + 强制 reflow 重放）。不再用元素级 backgroundColor 动画（区域偏小/无 bleed）。
2. **去掉「还剩 N 题未答」**：删除该提示（模板 + CSS + i18n `popup.unansweredRemaining`）。§3.5 阶段②不再显示计数。
3. **⌘↵ 与 ⌘] 行为必须一致**：⌘↵（`onCmdEnter`）改为复用与上一个/下一个同一套导航 `goToIdx`——
   **携带输入框焦点**到目标题（此前 ⌘↵ 不聚焦、⌘] 聚焦，不一致）。
4. **切题滚动位置被底部 footer 挡（疑输入框展开所致）**：`goToIdx` 改为**先置当前题不滚动 → 聚焦（触发折叠输入
   框展开、高度变化）→ 展开稳定后再滚动**，避免用展开前的高度算位置导致目标题露不全。
5. **提交按钮恢复 ⌘↵ 角标**：新增 `cmdEnterWillSubmit`（已看完 且 当前焦点之后再无未答题；焦点在末题恒真）
   ——`⌘↵` 角标 / 主按钮据此挂在提交上，恒等于 `onCmdEnter` 的实际动作（§3.5「谁挂 ⌘↵ = ⌘↵ 就干谁」不变，
   只是判据从「阶段③全答」放宽为「焦点之后无未答」，更贴近『到末题即可 ⌘↵ 提交』的直觉，且自由模式留空也能提交）。

> 相应地 §3.5 表格中「阶段②旁显『还剩 N 题』」作废；「阶段③才挂 ⌘↵」放宽为「`cmdEnterWillSubmit` 时挂 ⌘↵」。

### 2026-07（二轮实测反馈，已修——覆盖 §3.1/§3.2/§3.4 的若干原结论）

二轮真机连续实测后，用户对焦点/滚动/闪光/聚焦提出进一步要求，均已实现。以下结论**取代**前文对应条目：

1. **焦点来源去掉 hover（改 §3.1 F1）**：hover 会随滚轮把卡片移到光标下而乱跳，令人困惑。**移除 `@mouseenter`/`hovering`**，
   当前题（⌘1–9 落点）改由**滚动 scroll-spy + 点选/键盘导航**决定；hover 不再改焦点。
2. **按钮也闪（改 §3.2 B2 / §3.4 N2）**：`⌘[`/`⌘]`/`⌘↵` **和**「上一个/下一个」按钮**都**触发整题闪一下（统一移进
   `goPrev`/`goNext`，键盘处不再重复触发）。
3. **导航自动聚焦输入框（扩展 §3.4）**：用键盘或按钮切到某题时，`goToIdx` **总是**自动聚焦该题输入框（折叠框展开、
   光标进入），不再仅在「原本就有输入框聚焦」时才携带焦点。
4. **滚动统一「露出该题顶部」**：`scrollQuestionIntoView` 弃用比例定位（会把靠后的题只露个顶、导致再切时卡住），改为
   **顶部对齐 + 16px 呼吸位**（末题夹到底也完整可见）。目标题本就完整可见时不滚动（避免「看得见还硬滚一段」）。
5. **首题「上一个」两级露出（改 §3.4 N3）**：首题不再一律禁用『上一个』。在 Q2 按上一个/`⌘[` = 露出 Q1 顶部；站在
   **Q1** 再按上一个/`⌘[` = 把上方 message **完整滚回最顶**；已滚到最顶才禁用（`canGoPrev` = 纵向下 `!(首题 && atTop)`）。
6. **reveal-first（长 message 打开）**：刚打开时长 message 把 Q1 顶到屏外，点「下一个」/`⌘]`/`⌘↵` 应**先把当前题露出来 +
   聚焦 + 闪**，而非跳过它。判据用「**完全在屏外** `isCardOffScreen`」（仅底部被切一点不算，可正常推进）。
7. **防抢焦点**：对齐顶部后阅读线不落在目标题，故导航用 `NAV_LOCK_MS=700ms` 锁覆盖「聚焦展开(双 nextTick) + smooth
   动画」全程，动画结束后才交回 scroll-spy；用户**手动滚动**时才按 scroll-spy 切当前题。
8. **闪光区域几何**：`.q-card::before` 定为 `inset: 8px -6px -6px -9px`（上/右/下轻微外扩、**不覆盖题间分割线**；左侧
   外扩 -9px 留 ~7px 呼吸位、不贴窗口边框），accent 12% 淡底、圆角 8px、`opacity 1→0` 渐隐。
