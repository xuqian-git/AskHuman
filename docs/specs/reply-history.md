# 需求：回复历史记录（弹窗查看「我最近的回复」）

> 状态：已实现；后续搜索、来源上下文与 GUI Host 演进见下方实现补充。
> 关联计划：`docs/plans/reply-history.md`

> **当前实现补充（2026-07）**：历史窗口现由统一 GUI Host 承载并监听 `history.jsonl` 跨进程刷新；
> 搜索覆盖 message、题干、回答、附件、workspace、Agent、来源与渠道。记录新增可选 `agentKind`，普通 CLI
> 取调用环境，MCP 来源可在 daemon 异步探测后于 Coordinator 落盘前回填；旧记录缺字段时按 `source`
> best-effort 兼容。以下“独立窗口”指独立 WebView 窗口，不再要求独立进程。

## 1. 背景

`AskHuman` 每次调用弹窗（或经钉钉/飞书/Telegram 渠道）收集一次人类回应后即退出，**回应不留存**。用户希望在作答时能回看「我最近在当前项目里的回复」：包含当时的**原始 message / 问题**与**我给出的答案**，以及这条回复**是在哪个端（弹窗 / 钉钉 / 飞书 / Telegram）提交或取消的**——即使是经 IM 渠道回复的，也能在这里看到答案与渠道。

典型场景：用户正在弹窗里回答当前问题时，想对照**上一条**的 message 内容，因此希望历史是一个**可与弹窗并存的独立窗口**。

## 2. 目标

- 弹窗导航栏（置顶 / 切主题 / 设置 那一排）新增「历史」按钮，点击打开一个**独立的历史窗口**（类似设置窗，可与弹窗并存、便于对照）。
- 历史窗口展示**当前项目**最近的若干条回复记录；可经下拉切换到**其他项目**或**全部项目**。
- 同时支持命令行 `AskHuman --history` 独立打开历史窗口：默认展示**当前项目**（见 §3 项目识别）；附加 `--all` 则默认展示**全部项目**。
- 点开某条记录，看到**完整还原**当时提问的界面（消息 + 附件 + 各题选项 + 我填写的回复 + 图片 + 文件），但为**只读**（输入框已是填写完成态、不可编辑、无提交按钮），并展示**状态信息**（由哪个端提交 / 取消、时间等）。
- 记录覆盖**所有渠道**：无论回复发生在弹窗还是 IM 渠道，都能在历史里看到答案与对应渠道。
- 历史条数**可在设置中配置**（默认 200，0 = 停止新增记录但仍可查看旧记录）。
- 提供「清空历史」（可选当前项目 / 全部）。
- **全平台**可用。

## 3. 已确认决策

### 3.1 项目识别（历史过滤的基础）

| 编号 | 决策项 | 结论 |
|---|---|---|
| P1 | 「当前项目」如何确定 | 以 `AskHuman` 被调用时的**工作目录 cwd** 为起点，**向上探测第一个含 `.git` 的目录**作为项目根；找不到则回退 cwd。规范化为绝对路径作为项目 **key**，其 basename 作为**显示名**。 |
| P2 | 项目信息如何到达记录点 | 由 **CLI 计算**项目 key（见 P1），随任务上送 Daemon，并下发给弹窗 / 历史窗（**revisit 既有 A11「不传 cwd」决策**：本需求需要项目维度，故新增此上下文字段）。单进程回退路径与独立窗口在各自进程内直接计算。 |
| P3 | `--history` / `--history --all` | `--history` 默认当前项目；`--history --all` 默认全部项目；窗口内仍可用下拉切换。 |

### 3.2 记录内容与时机

| 编号 | 决策项 | 结论 |
|---|---|---|
| R1 | 汇聚点 | 所有渠道、所有运行模式的终态结果都汇聚到「每个请求的协调器 `Coordinator.finish()`」（其内调用 `render_result`）。**在此处记录**，天然覆盖弹窗 / 钉钉 / 飞书 / Telegram / 单进程 / headless。 |
| R2 | 每请求一条 | 一次提问产生**一条**记录（取首个生效的终态结果，即抢答赢家的整套结果）。 |
| R3 | 记录范围 | 记录「**发送**」（含空回复 = 确认继续、多题部分未答 / 全未答）与「**用户主动取消**」（关弹窗 / IM 端取消）。**不记录系统性取消**（超时 / 连接断开 / `daemon stop`）——这些走 `cancel_request`、本就无 result，自然不进历史。 |
| R4 | 记录字段 | 每条含：请求 id、时间戳、项目 key、来源名（`ASKHUMAN_ENV_SOURCE_NAME`）、**渠道**（提交 / 取消端：popup/dingding/feishu/telegram）、动作（send/cancel）、是否 markdown、原始 message（文本 + `-f` 附件路径）、各问题（题干 + 选项）、各题作答（所选项、文本、**图片路径**、**文件路径**）。 |
| R5 | 图片 / 文件 | **只存路径**，不存 base64。展示时 best-effort 读取；文件不存在（如临时图片被 24h 清理后）显示**占位 / 不可用**，不报错。 |

### 3.3 存储与容量

| 编号 | 决策项 | 结论 |
|---|---|---|
| S1 | 落盘位置 / 格式 | `~/.askhuman/history.jsonl`：每行一条 JSON，**追加写**。读取时逐行解析，坏行跳过。 |
| S2 | 容量上限（可配置） | 新增设置项 `general.historyLimit`：默认 **200**。裁剪对 0 不再特判：`>0` 每次写入后裁剪到最近 N 条；`=0` **停止新增**，但仍按相同时机把已有记录**裁剪到 0（即清空）**。 |
| S3 | 设置页降低条数的提示 | 设置页填入的新条数若**小于现有记录条数**（含填 0），在该项下方显示一行小字：「已有日志超过该条数，将在下次调用 AskHuman 时清理」，并提供一个**链接形式的「立即清理」按钮**，点击即把历史裁剪到该条数。 |
| S4 | 并发安全 | 多进程 / 多请求可能同时写：写入 / 裁剪 / 清空走**文件锁 + 原子写**（unix `flock`；非 unix 走原子 rename 兜底）。读取容错。 |

### 3.4 界面与交互

| 编号 | 决策项 | 结论 |
|---|---|---|
| U1 | 入口 | 弹窗导航栏在「置顶 / 主题 / 设置」一排新增「历史」按钮（钟表 / 回溯类图标）。 |
| U2 | 窗口形态 | **独立窗口**（与设置窗同机制：同进程内新建、复用主题与玻璃 / 毛玻璃样式），可与弹窗并存以便对照。 |
| U3 | 布局 | 顶部：项目过滤下拉（默认当前项目，可切「其他项目」/「全部」）+ 清空入口。主体：**左侧列表**（按时间**倒序**）+ **右侧详情**。 |
| U4 | 列表项 | 显示：相对时间、**渠道徽标**、动作状态（已提交 / 已取消）、消息或首题题干摘要；过滤为「全部 / 其他项目」时额外显示项目名。 |
| U5 | 详情还原度 | **完整还原**当时提问 UI：消息（按 markdown 渲染）+ `-f` 附件区 + 每题（题干 + 选项，已选项高亮、不可点）+ 我填写的回复（文本以只读形式呈现）+ 图片缩略图 + 回复文件胶囊。**无任何编辑 / 提交能力**。 |
| U6 | 状态信息 | 详情顶部展示状态横幅：动作 + 渠道 + 时间（如「由 钉钉 提交 · 6月7日 15:20」/「由 弹窗 取消 · …」）；可附项目名 / 来源名。多题中未作答的题显示「未作答」。 |
| U7 | 清空 | 提供「清空历史」，可选**当前项目**或**全部**（二次确认）。 |
| U8 | 详情实现 | 另建**独立的只读渲染组件**，复用同一套视觉 / CSS，但**不改动现有 `PopupView.vue`**（隔离回归风险，二者独立演进）。 |

## 4. 约束与既有规则（不可破坏）

- **stdout 洁净 / 退出码 / 结果区块格式**：历史记录为**旁路**行为，绝不改变给调用方的 stdout、退出码（0/1/3）、结果区块语义；记录失败必须**静默兜底**（不影响主流程）。
- **既有契约不变**：多渠道抢答、图片回传落盘、配置容错、`--settings`/`--help`/`--version`、Daemon/IPC 协议向后兼容（仅**新增**字段，旧端忽略未知字段、缺字段走默认）。
- **不改动 `PopupView.vue` 的现有模板 / 逻辑**（仅在其导航栏新增一个按钮触发打开历史窗口）。
- 配置文件：`historyLimit` 缺失走默认 200；未知字段忽略。
- 注释与提交信息用英文；界面文案中英双语（沿用 i18n）。

## 5. 验收标准

1. 弹窗导航栏出现「历史」按钮；点击打开独立历史窗口，默认展示**当前项目**记录，且可与弹窗并存对照。
2. 在弹窗回复后再次打开历史，能看到刚才那条记录；经钉钉 / 飞书 / Telegram 回复的记录亦出现并标注对应渠道。
3. `AskHuman --history` 打开窗口默认当前项目；`--history --all` 默认全部项目；窗口内下拉可切换项目 / 全部。
4. 点开记录：完整还原提问 UI（只读、无提交）、展示状态横幅（提交 / 取消渠道 + 时间）；多题部分未答的题显示「未作答」。
5. 图片 / 文件 best-effort 展示；缺失（如临时图片已清理）显示占位、不报错。
6. 设置 `historyLimit`：默认 200；设为 0 后停止新增并清理已有记录（与 >0 同时机：下次调用 `AskHuman` 或点「立即清理」时裁剪）；超额（含填 0）时显示提示 + 「立即清理」可即时裁剪；正常调用 `AskHuman` 时自动裁剪到上限。
7. 仅记录用户主动取消；系统性取消（超时 / 断连 / `daemon stop`）不进历史。
8. 「清空历史」（当前项目 / 全部）生效。
9. 全平台均可记录与查看；记录失败不影响主流程（stdout / 退出码不受影响）。
