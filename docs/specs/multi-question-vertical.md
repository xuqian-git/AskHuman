# 需求：多问题弹窗改「纵向列表」

> 状态：已实现，由 `experimental.verticalQuestions` 门控，默认关闭。
> 关联计划：`docs/plans/multi-question-vertical.md`
> 关联既有：`docs/specs/multi-question.md`（多问题基础能力）

## 1. 背景

当前多问题（n>1）弹窗一次只展示一题，靠「上一个 / 下一个」切换、全部查看过后才出现提交（见 `docs/specs/multi-question.md` §3.3）。本需求把多问题弹窗改为**所有问题纵向平铺**：一次性把全部问题竖向列出，用户滚动浏览，同时**保留**上一个 / 下一个按钮（语义改为滚动定位）。**单问题（n=1）保持现状不变**。仅前端改动（`src/views/PopupView.vue`）。

## 2. 目标与总体形态

- 共享 Message（描述 + 附件）仍常驻最上方。
- 其下把**所有问题纵向平铺**，每题一个卡片：`Question i/n` 头部 + 分割线 + 题干 + 选项 + 补充输入框 + 该题图片/回复文件。
- 取消原「单题滑动切换」动画与 `current` 单题渲染。
- 底部仍有取消 / 上一个 / 下一个；**最后一题被看到后**出现发送按钮。

## 3. 已确认决策

### 3.1 「已看到」判定与发送时机

| 编号 | 决策项 | 结论 |
|---|---|---|
| V1 | 「已看到」判定 | 一个问题卡片的**底部进入视口**即视为「已看到」（用卡片底部哨兵 + IntersectionObserver），**兼容超长问题**（卡片高于视口也能判定）|
| V2 | 发送出现时机 | **最后一题被「看到」**后出现发送按钮；若问题少、一屏全可见，则进入即出现 |
| V3 | 键盘前进也算看到 | 「已看到」= 卡片底部进视口（滚动）**或**通过键盘把该题设为过「当前题」；故键盘走到末题即出现发送 |

### 3.2 上一个 / 下一个（导航）

| 编号 | 决策项 | 结论 |
|---|---|---|
| N1 | 「当前题」指针 | 引入与「输入框聚焦」解耦的**当前题**指针；= 最后一个完整可见的问题；初始 = 第 1 题 |
| N2 | 下一个 | 当前题指针下移一题，并把目标题**滚动到刚好完整可见**（不强制贴顶）|
| N3 | 上一个 | 当前题指针上移一题，同样滚到刚好完整可见 |
| N4 | 边界 | 首题禁用「上一个」；当前题到末题后「下一个」禁用、出现发送 |

### 3.3 「当前题」高亮与快捷键

| 编号 | 决策项 | 结论 |
|---|---|---|
| H1 | 高亮 | **当前题**有淡色高亮（具体效果实现后由用户确认）；一眼看出快捷键作用在哪题 |
| H2 | 选项角标 | **仅当前题**的选项显示 `⌘1–9` 角标，其余题不显示 |
| H3 | 初始焦点 | 多问题模式**不自动聚焦**任何输入框（保持全部折叠便于浏览）；单问题维持自动聚焦 |
| H4 | 打字切当前题 | 在某题输入框打字 / 聚焦 → 当前题自动切到该题 |

### 3.4 全键盘流程（复刻「选项→下一个→选项→下一个」）

| 编号 | 决策项 | 结论 |
|---|---|---|
| K1 | `⌘1–9` | 勾选**当前题**对应选项 |
| K2 | `⌘↵` | 当前题非末题 → 前进到下一题（指针下移 + 滚动可见 + 标记看到）；当前题为末题 → 提交 |
| K3 | `⌘[` / `⌘]` | 当前题上移 / 下移（同样滚动到可见）|
| K4 | `⌘W` | 取消（沿用二次确认）|
| K5 | `⌘D` 语音 | 作用于**当前题**输入框 |

> 纯键盘可走：`⌘1`（选 Q1）→ `⌘↵`（到 Q2）→ `⌘3` → `⌘↵` … → 末题 `⌘↵` 提交。

### 3.5 折叠输入框（仅多问题模式）

| 编号 | 决策项 | 结论 |
|---|---|---|
| C1 | 默认折叠 | 每题补充输入框默认 **1 行高** |
| C2 | 展开条件 | **聚焦时**或**已有内容时**展开到现有（多行）高度 |
| C3 | 收回条件 | **失焦且内容为空** → 收回 1 行 |

### 3.6 图片归属（粘贴 / 拖拽 / 按钮）

| 编号 | 决策项 | 结论 |
|---|---|---|
| I1 | 拖拽 | 按**落点所在的问题卡片**归属（drop 目标 → 最近的题卡片）|
| I2 | 粘贴 | 按**焦点位置**：某题输入框聚焦 → 该题；都没聚焦 → 归**当前题** |
| I3 | 按钮 | 每题输入框内置独立「添加图片」按钮 → 加到该题 |
| I4 | 语音 | 每题各有麦克风，作用于该题；`⌘D` 作用于当前题 |

### 3.7 其它沿用

| 编号 | 决策项 | 结论 |
|---|---|---|
| O1 | 提交校验 | 沿用现状：非严格模式可空答；严格选择(select-only)每个有选项的题需至少选一个 |
| O2 | 取消二次确认 | 沿用现状（有任一回答时二次确认）|
| O3 | 输出契约 | 不变（聚合输出由后端负责，见 `docs/specs/multi-question.md` §3.2）|
| O4 | 单问题 | n=1 完全保持现状（无纵向列表、无当前题高亮、维持自动聚焦与现有底部）|

## 4. 约束与既有规则（不可破坏）

- 仅前端改动（`src/views/PopupView.vue`）；不动数据模型 / 提交协议 / 后端聚合输出。
- 单问题路径行为逐项保持现状（焦点、输入框高度、底部布局、键盘）。
- 不引入新依赖。

## 5. 验收标准

1. 多问题弹窗一次纵向列出全部问题，每题有 `Question i/n` 头部 + 分割线。
2. 滚动使某题底部进视口 → 该题标记「已看到」；滚到末题底部 → 出现发送。
3. 「当前题」有可见高亮；仅当前题显示 `⌘1–9` 角标。
4. 纯键盘：`⌘1` 选项 → `⌘↵` 下一个 …→ 末题 `⌘↵` 提交；`⌘[`/`⌘]` 上/下一个；`⌘W` 取消。
5. 多问题输入框默认 1 行，聚焦或有内容展开，失焦且空收回。
6. 拖拽图片落到某题归该题；粘贴归焦点题（无焦点归当前题）；每题「添加图片」按钮归该题。
7. 上一个 / 下一个滚动到目标题刚好完整可见；首题禁用上一个、末题后下一个禁用。
8. 单问题(n=1) 行为与现状完全一致。

### 2026-07（回看 Message 时固定输入框）

`docs/specs/popup-pinned-composer.md` 对 §3.5 增加一条局部覆盖：用户激活某题输入框后向上回看、原输入位置落到视口下方时，输入框停靠在底部；停靠期间即使失焦且为空也保持展开。它重新回到原位后，既有“失焦且为空则折回一行”规则继续生效。composer owner 与本规格的当前题指针相互独立，滚动更新当前题不会改变停靠输入的答案归属。
