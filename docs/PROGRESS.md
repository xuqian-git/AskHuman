## 后续增强 / 性能优化

### 钉钉卡片「变灰」延迟（跨渠道抢答路径）

- **背景**：跨渠道抢答的「已在 X 回答」置灰走 OpenAPI `updateCard`/`patchCard`（卡片回调即时空 ACK 以满足 3 秒约束）；卡片**自提交**路径已改同步回包即时置灰。
- **影响**：跨渠道置灰是一次独立 HTTPS 调用，较单进程同步回包慢约 100–300ms（仅视觉延迟，功能一致）。
- **可选优化**：让会话经 oneshot 把回包回传 Router 的 Reader、在 3 秒内用长连接写回（代价：Reader↔会话耦合、读循环需短暂等待）。当前判断收益有限，暂不做。
