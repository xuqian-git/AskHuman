# 钉钉渠道配置

简体中文 | [English](./dingtalk-setup.en.md)

本文说明如何创建并配置一个钉钉「企业内部应用 + 机器人」，使 AskHuman 的「钉钉 Channel」可用。该渠道采用 **企业内部应用 + 机器人 + Stream 模式（WebSocket 长连接）+ 单聊** 形态，**无需公网 / 域名 / 证书**即可收发消息与卡片回调。提问以互动卡片逐题下发，你在卡片里勾选、补充文字并提交即可作答。

> 前提：你在钉钉组织内有开发者权限，可创建企业内部应用（若组织尚未在开放平台完成企业认证，需先认证）。

## 一、创建应用并启用机器人

有两种方式，**推荐方式一**：用钉钉提供的「一键创建」入口，开箱即用、无需手动开通权限。只用方式一的话，可直接跳到「二、发布与可用范围」。

### 方式一（推荐）：OpenClaw 一键创建，免配权限

钉钉开放平台提供了「一键创建 OpenClaw 机器人应用」入口（OpenClaw 是钉钉上的一款 AI 助手；该入口创建的是一个**标准的 Stream 模式机器人内部应用**，AskHuman 可直接复用）。它会自动建好应用、启用机器人、设为 Stream 模式，并预置互动卡片与单聊消息所需的接口权限，因此**通常开箱即用、无需再手动开通权限**。

1. 进入 [钉钉开放平台开发者后台](https://open-dev.dingtalk.com) → 顶部「应用开发 → 钉钉应用」。
2. 在「钉钉应用」页顶部找到「一键创建 OpenClaw 机器人应用」提示栏，点右侧「**立即创建**」。
3. 在弹窗中填写机器人名称 / 简介 / 图标（图标可用仓库内置头像，见下），点「确定」。
4. 创建完成后复制 **ClientId（AppKey）** 与 **ClientSecret（AppSecret）**，稍后填入 AskHuman。

该入口已自动启用机器人、设为 Stream 模式并预置所需权限，**无需再开通权限**；接着按「二、发布与可用范围」确认应用已发布、可用范围包含你即可。

### 方式二：手动创建企业内部应用

1. 进入 [钉钉开放平台开发者后台](https://open-dev.dingtalk.com) → 创建「企业内部应用」，记录应用凭证 **ClientId（AppKey）** 与 **ClientSecret（AppSecret）**。
2. 在「应用能力」中添加并启用 **机器人**；机器人 `robotCode` 即应用 AppKey，无需单独配置。
3. 在机器人配置里把「消息接收模式」选为 **Stream 模式**（推送到本地长连接，零公网回调）。
4. 按下方「开通权限」开通所需接口权限。
5. 在「版本管理与发布」发布版本（见「二、发布与可用范围」）。

#### 开通权限

> 仅手动创建需要；方式一（OpenClaw 一键创建）的应用已预置以下权限。

钉钉的权限需在应用「权限管理」中**逐项搜索并申请**（不支持像飞书那样的 JSON 批量导入）；申请后免审批、即时生效，但**变更权限后需重新发布版本**才会生效。按本渠道实际调用的接口，需开通：

| 需开通的权限 | 用途 | 涉及接口 |
| --- | --- | --- |
| **企业内机器人发送消息权限**（`qyapi_robot_sendmsg`） | 给用户单聊发送 文本 / 图片 / 文件 / 卡片；下载用户发来的图片 / 文件（人 → AI 附件回传） | `robot/oToMessages/batchSend`、`robot/messageFiles/download` |
| **`Card.Instance.Write`**（创建和投放卡片实例） | 下发互动卡片高级版，并在收尾 / 抢答时更新卡片 | `card/instances/createAndDeliver`、`card/instances`（更新） |

说明：

- **媒体上传**（发送 `-f` 附件前先上传取 mediaId）走基础权限 `qyapi_base`，企业内部应用**默认开通、无需申请**。
- 在「权限管理」搜索框分别输入「企业内机器人发送消息权限」与「Card.Instance.Write」即可定位并申请。

> 头像：仓库内置两张可作机器人头像的图片（深色 / 浅色背景），也可用于其它场景：[深色](../../assets/avatars/bot-avatar-dark.jpg) · [浅色](../../assets/avatars/bot-avatar-light.jpg)。

## 二、发布与可用范围

1. 在「版本管理与发布」中创建版本并发布（机器人能力、Stream 模式、权限均需**发布后**才生效）。
2. 确认机器人的**可用范围**包含目标用户，否则发送会报「机器人对该用户无可用范围」类错误。

## 三、（可选）自定义卡片模板

AskHuman 默认使用**内置的高级版互动卡片模板**，开箱即用、无需配置。如需自定义卡片样式，可在钉钉卡片平台搭建并发布一个**高级版**互动卡片模板（同一应用），把模板 ID 填入设置页的「卡片模板 ID」；留空即用内置默认模板。

## 四、在 AskHuman 中填写

打开设置页 → 「通信渠道」→「钉钉」，开启开关后填写：

| 字段 | 说明 |
| --- | --- |
| ClientId | 应用 AppKey |
| ClientSecret | 应用 AppSecret |
| UserId | 接收 / 作答用户的 userId（单聊）。可点「自动识别」：先校验 ClientId/ClientSecret，再提示你用目标账号私聊机器人发送一个 4 位数字以精确回填 |
| 卡片模板 ID | 留空用内置默认模板；填入则用你自建的高级版卡片模板 |

填好后点「测试连接」：会换取 token 并向该 userId 的单聊发送一条测试消息，钉钉内收到即配置成功。

## 五、交互与回退行为

- 提问以**互动卡片（高级版）**逐题下发：卡片内勾选预定义选项（多选）、可补充文字，点「提交」完成该题（回调走 Stream，零公网）。
- 作答期间在聊天里发送的**图片 / 文件**会被累积进该题答案（图片进 `[图片]`、文件进 `[文件]`）；纯文字请用卡片输入框补充。
- 若卡片投放失败，会自动**回退**为「纯文本 + 编号选项」：回复编号（多选用逗号，如 `1,3`）/ 直接输入文字 / 发送图片 / 文件均可作答。
- Message 的附件（`-f`）会经媒体上传后随 Message 发送。
- 多渠道同启时以「整个会话」为粒度**抢答**：哪端先答完全部题即采用该端结果，其余自动收尾。

## 六、常见问题

| 现象 | 可能原因 / 解决 |
| --- | --- |
| 测试连接失败 | ClientId / ClientSecret 填错或有多余空格；或应用尚未发布 |
| 收不到卡片、自动回退成纯文本 | 手动创建的应用缺 `Card.Instance.Write`（见「方式二 · 开通权限」）；或机器人未启用 / 未发布 |
| 发送或下载图片 / 文件报权限错误 | 手动创建的应用缺「企业内机器人发送消息权限」（见「方式二 · 开通权限」） |
| 机器人单聊收不到消息 | 消息接收模式不是 Stream；或应用未发布；或目标用户不在可用范围内 |
| 报「机器人对该用户无可用范围」 | 目标用户不在机器人可用范围内（见「二、发布与可用范围」） |
| 「自动识别」收不到验证码 | 需用**目标账号**私聊机器人发送提示的 4 位数字；并确认机器人已发布且对你可用 |
| Windows 上并发提问偶发回复串台 | 钉钉同一应用同一时刻仅允许**一条** Stream 长连接；Windows 端目前并发发起多个提问时，多条 Stream 可能相互抢消息，建议避免对同一应用并发提问 |
